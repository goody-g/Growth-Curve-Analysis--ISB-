gPrepareDatabaseR Documentation

Prepares a Pre-Database File

Description

Prepares a pre-database file with well number, label, date and intials information to be completed by user. See manual.

Usage

gPrepareDatabase("FYL", 20080422)

Arguments

initials The initials of the experimenter, which must match with the initials on the layout file and experiment folder and be consistent with past experiments.
date The date chosen to represent the experiment, which must match with the date on the layout file and experiment folder.
control An object of class "gControl" which provides options for gPrepareDatabase such as the root folder of the database.

Details

This function simplifies adding a database entry to the growth curve database associated with the GCAF package.

The layout file and results file (for formatting see documentation) are saved in the control$root.folder location at Database/Layouts/LayoutII########.txt and Database/########II/ResultsII########.txt where II are the initials and # is the date.

Running gPrepareDatabase will then create a barebones database file in the folder with the results file which should be edited using Excel or similar.

The gPrepareDatabase will not write over existing database files, so delete the original manually if you wish to replace.

Value

Database A basic database file is saved in the appropriate folder for further editing with a spreadsheet editor (Excel).

Examples

# After having created the layout file and experiment folder in the appropriate locations (see documentation):
gPrepareDatabase(initials="ST", date=20090511)

#Then open the database file and complete information using Excel or a similar spreadsheet editor.
